# Changelog - PPDS.Cli

All notable changes to PPDS.Cli will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- **`ppds webresources create <file>`** — create a brand-new web resource directly from a local file: type inferred from the file extension (`--type` single-type override), `--display-name`, optional atomic `--solution` binding (SDK equivalent of the `MSCRM.SolutionUniqueName` header), and `--publish`. Fails with `WebResource.AlreadyExists` (hinting at `update`) when the name is taken (#1207).
- **`ppds webresources update <name|id> <file>`** — replace an existing web resource's content from a local file (binary types allowed — this is file replacement, not text editing), with optional `--publish`. Resolves by exact name or GUID only, so a fuzzy match can never overwrite the wrong resource; warns when the file extension implies a different type than the stored resource (#1207).
- **Canonical metadata nouns** `entity`, `attribute`, `optionset` replace `table`, `column`, `choice` as the primary command surface. Old nouns remain as deprecation shims that delegate to the same execute path and print a migration hint to stderr (#1159).
- **Status reason management** — `ppds metadata entity add-statusreason|list-statusreasons|update-statusreason|remove-statusreason` for first-class statuscode option-value management (#1160).
- **`OptionValueDeriver`** shared pure helper for publisher-prefix-based option value derivation — explicit-value wins, prefix×10000 base, gap-fill over existing values (#1160).
- **`ppds forms`** — form (systemform) management: list/get, set-xml with schema validation, and structured tab/section/field/sub-grid editing on Main forms, with --publish/--solution side effects (#1163).
- **`ppds views`** — savedquery (view) management: list/get, add/remove/update/reorder columns, set/clear sort, set/clear filter, and set-fetchxml (#1162).
- **`ppds model-driven-app`** — app navigation management: list, get, sitemap, set-sitemap-xml, add-table/remove-table, and set-forms/views/charts, with bundled sitemap XSD validation (#1165).
- **`ppds model-driven-app add-copilot|remove-copilot|list-copilots`** — wire a Copilot Studio agent (bot) into an app. The binding is an `appelement` whose polymorphic `objectid` targets the `bot` table, set via the SDK `EntityReference("bot", id)`. The Dataverse Web API cannot create this binding: `appelement.objectid` has three targets (`bot`, `aiskillconfig`, `mcpserver`) that all share the `objectid` navigation property, so a bare `@odata.bind` mis-resolves to `aiskillconfig` and `objectid_bot@odata.bind` is undeclared — the SDK carries the explicit target type and has no such ambiguity. `objectid` is create-only (the platform ignores updates to it), so `add-copilot` always creates the binding and falls back to a unique-suffixed `uniquename` when the maker-convention name is already taken by a stale row. Supports `--dry-run` and `--publish`. Note: for the model-driven app designer to surface the agent in-app, the target bot must be a valid app-assistant (`isLightweightBot`); a bot created standalone in Copilot Studio binds correctly at the `appelement` level but won't render until configured as an app assistant (#1185).
- **`ppds model-driven-app inspect-app-assistant`** — read-only "copilot doctor" that inspects an app's Copilot (bot) `appelement` bindings and reports problems without writing anything: bots bound but not configured as app assistants (`isLightweightBot=false`, so they never render in-app), duplicate `objectid → bot` bindings for the same app, and orphan copilot-shaped `appelement` rows with a null `objectid`. Each finding reports the bot display name, bot id, `isLightweightBot`, the offending appelement id(s), and a one-line remediation hint. Emits data to stdout (table or `--json`) and status to stderr, and exits non-zero when any problem is found so it is scriptable in CI / pre-publish checks (#1193).
- **`ppds api request`** — authenticated raw Web API calls to Dataverse (GET/POST/PATCH/DELETE) with default OData headers and production write protection (#1164).
- **`ppds api request --profile|-p`** — target a specific authentication profile, consistent with the other commands (previously `api request` could only switch the environment, not the profile).
- **Local Choice columns** — `ppds metadata attribute create --type Choice` with inline --option/--options-file, plus `ppds metadata attribute add-option|update-option|remove-option` for column-scoped option sets (#1161).
- **`ppds metadata optionset remove-option --label`** — target an option on a global option set by label as an alternative to `--value` (mutually exclusive), matching `attribute remove-option`. Resolution happens in the authoring service against unpublished metadata (so just-added options are targetable before publish); unresolved targets report `OPTION_NOT_FOUND` (#1169).
- **`--publish` on `ppds metadata attribute update`, `entity create`, `entity update`** — publish the affected entity immediately after the change, matching the other authoring verbs; without it the existing publish-hint behavior is unchanged (#1171).
- **`--dry-run` on `ppds metadata optionset add-option|update-option|remove-option`** — validate without persisting, matching the optionset CRUD verbs and the attribute option commands. `update-option`/`remove-option --dry-run` verify the target option exists, and `remove-option --dry-run` skips the confirmation prompt (#1172).
- **`ppds schema compare`** — detect schema drift between a CMT data package and a target environment (`--data` / `-e|--environment`), or between two live environments (`-se|--source-env` / `-te|--target-env`), before an import fails. Backed by the pure `SchemaComparisonService` over a Dataverse/CMT-decoupled `SchemaSnapshot` (`EnvironmentSnapshotLoader` for live envs, `DataPackageSnapshotLoader` for CMT zips). Emits a severity-categorized report (Error/Warning/Info) as text or JSON to stdout, with the exit code reflecting the highest severity (0/1/8) so it is CI-scriptable (#42, #1060).
- **`ppds metadata entity update --icon-small|--icon-medium|--icon-vector`** — assign entity icons (web-resource logical names) via the SDK `UpdateEntityRequest`; the Web API `EntityDefinitions` PATCH silently rejects icon updates, so these route through the SDK path (#1244).
- **`--raw` / `--unpublished` on `ppds model-driven-app sitemap`** — `--raw` emits the sitemap XML verbatim (JSON mode returns `{ xml }`); `--unpublished` reads the draft sitemap (#1242).
- **`--unpublished` on `ppds forms get` and `ppds views get`** — opt into the draft (unpublished) record; the default remains the published version, matching `webresources get` (#1243).
- **`ppds data export --format json`** — export to a single-file PPDS-native JSON payload (`Cmt` remains the default, so existing pipelines are unaffected). JSON mirrors the CMT XML structure (schema metadata, per-entity typed records, M2M associations). File-column binaries are not serialized in JSON v1 — use CMT format if files must round-trip (#1059).

### Changed
- **`ppds metadata entity` authoring verbs accept a positional `<entity>`** — `update`, `delete`, `add-statusreason`, `list-statusreasons`, `update-statusreason`, and `remove-statusreason` now take the entity logical name positionally (`ppds metadata entity update hsl_veterinarian --solution …`), matching how the parent `entity <name>` read lookup is typed. `--entity` keeps working as a fully equivalent flag; if both are supplied they must agree or the command errors clearly instead of silently picking one (#1208).
- `ppds metadata table|column|choice` are now deprecated shims; use `entity|attribute|optionset` instead.
- **`ppds metadata optionset update-option`** now matches the `attribute update-option` calling convention: `--value` OR `--label` (mutually exclusive) selects the target option, `--new-label` carries the new label, and optional `--color` sets a hex color. Previously `--value` was the required selector and `--label` was the required new label; scripts using the old shape must move the new label to `--new-label`. The deprecated `choice update-option` shim keeps the legacy shape (#1170).
- **`ppds api request --path`** now detects when Git Bash/MSYS has rewritten a leading-slash path into a Windows path (e.g. `/api/data/v9.2/contacts` → `C:/Program Files/Git/api/data/v9.2/contacts`) and augments the "Path must start with '/'" error with a hint to prefix `MSYS_NO_PATHCONV=1`, set `MSYS2_ARG_CONV_EXCL='*'`, or run from PowerShell/cmd. The command help also documents the gotcha (#1204).

### Fixed
- **`ppds metadata optionset update-option|remove-option --label` no longer silently targets the wrong option on duplicate labels** — duplicate option labels are legal in Dataverse, but the authoring service resolved a `--label` selector with a first-match lookup, so `update-option`/`remove-option` could mutate or delete whichever matching option happened to come first. Label resolution now collects every match and, when more than one option shares the label, throws `MetadataAuthoring.AmbiguousOptionLabel` listing all matching values and directing the user to disambiguate with `--value`. The same guard is applied to the attribute-local choice path (`attribute update-option|remove-option --label`) and the status-reason paths (`entity update-statusreason|remove-statusreason --label`). Single-match and no-match (`OPTION_NOT_FOUND`) behavior is unchanged (#1235, regression from #1229).
- **`ppds forms set-xml` now completes the modern-form round-trip** — after #1203/#1206 unblocked the schema gate, the very next gate (`FormXmlValidator.ValidateGuids`) still rejected real Microsoft model-driven forms with `Form XML contains a non-brace-format GUID 'fullname' on attribute 'id'`. The custom check was wrong two ways: its logical-name exemption covered only `<control>.id`, but real forms also carry logical names on `<data>.id` (e.g. `fullname`) and `<dependency>.id` (e.g. `absoluteurl`); and it demanded brace-wrapped GUIDs, while real forms use valid *unbraced* GUIDs (e.g. on `<section>.id`). The bundled XSD already validates GUID format correctly — `FormGuidType` accepts braced and unbraced GUIDs (`\{?...\}?`) and types logical-name ids as `xs:string` — so the hand-rolled format check was a redundant reimplementation that had drifted out of sync with the schema. It is removed; the XSD is now the single source of truth for format, and `ValidateGuids` keeps only the one thing the schema cannot express: cross-element GUID uniqueness (over GUID-valued ids, brace-normalised). Completes the modern-form `set-xml` round-trip started by #1203/#1206 (#1209).
- **`ppds forms` write commands no longer fail on modern UCI forms** — `forms set-xml` and the structured `add/update/remove tab|section|field|subgrid` editors re-validate the whole form document against the bundled `FormXml.xsd` before writing to Dataverse. The root `<form>` element's type was a closed attribute allowlist with no `xs:anyAttribute`, so the `headerdensity` attribute Microsoft added to modern model-driven (UCI) Main forms (e.g. `headerdensity="HighWithControls"`) was rejected with `The 'headerdensity' attribute is not declared`. The schema now declares `headerdensity` and accepts unknown `<form>` attributes leniently (`xs:anyAttribute processContents="lax"`) so future server-side additions no longer hard-fail valid forms. GUID/control-id validation is unchanged (#1203).
- **Reliable view writes** — `ppds views set-fetchxml|set-filter|clear-filter|set-sort|clear-sort|reorder-columns|update-column|remove-column` no longer fail opaquely or report a false success, and now succeed on managed views. Every fetchxml- and layoutxml-mutating command routes through one shared verified write path that (a) surfaces the underlying Dataverse fault (e.g. `0x80040216`) instead of the bare `Details: setting fetchxml`; (b) echoes `returnedtypecode` into the savedquery write so the platform applies fetchxml changes — the missing entity context was the actual cause of `0x80040216`; (c) verifies the change persisted via a lag-tolerant read-back against the *unpublished* record (`RetrieveUnpublishedMultiple`), so a draft write is no longer falsely reported as `View.UpdateNotPersisted`; and (d) reserves the `View.ManagedComponentNotEditable` error for genuinely locked components (`iscustomizable = false`) instead of refusing all managed views. Live-verified against a managed Quick Find view (#1190, #1194, #1200, #1201).
- `ppds model-driven-app set-views|set-forms|set-charts` with explicit named components no longer fail with `Input field type 'EntityReference' does not match expected type 'Guid' for field 'AppId'`. The shared `AddAppComponents` helper now passes the appmodule id as a `Guid` (matching the working `RemoveAppComponents` sibling); a regression test asserts the request parameter type (#1183).
- **`ppds forms`/`ppds views` mutations no longer silently overwrite earlier unpublished edits** — every mutation now reads the *unpublished* record (`RetrieveUnpublishedMultiple`) before writing, so successive edits compose instead of the later write clobbering the earlier one off a stale published read. Forms additionally detect non-customizable forms up front and throw `FormNotCustomizable` (Dataverse accepts the update but never persists it), and layout mutations against views with no `layoutxml` throw a clear `View.NoLayout` instead of a cryptic `0x80040216` (#1243).
- **`ppds model-driven-app add-table`/`remove-table` now work on draft (never-published) maker apps** — the sitemap is resolved by `sitemapnameunique == appmodule.uniquename` (and reads use `RetrieveUnpublishedMultiple`), so edits patch the maker's existing sitemap instead of minting a second one that Dataverse rejects with `0x80050111` ("App can't have multiple site maps"). Table membership is also registered under the table's own logical name rather than the literal `"entity"`, which had registered a spurious "Entity" component (#1242).
- **`ppds metadata attribute create --type DateTime` with `DateOnly` behavior no longer rejected** — `BuildDateTimeAttribute` now sets `Format=DateOnly` automatically when the behavior is `DateOnly` (Dataverse rejects a `DateOnly` behavior paired with the previous default `DateAndTime` format) (#1244).
- **`ppds publish` surfaces environment-lock faults with actionable guidance** — when Dataverse rejects a publish because another import/publish is already running, the error now states that your edits were saved and to retry once the environment is idle, instead of a raw fault. `ppds publish --type entity --solution <name>` without explicit entity names is also rejected up front with a clear message (#1245).
- **Option colors now persist** — color writes route through `OptionMetadata.Color` re-sent via `UpdateOptionSet` (global choices) / `UpdateAttribute` (local column options and status reasons) instead of the `request["Color"]` indexer, which the Insert/Update OptionValue SDK messages define no parameter for and silently drop. Local-option color updates send a minimal carrier attribute so unrelated attribute-level managed properties (e.g. `RequiredLevel`) are not reset (#1233, #1234).
- **`ppds metadata column update` now persists `RequiredLevel` changes and signals the publish requirement** — the service sends a fresh, minimal `AttributeMetadata` (copying only the requested fields) instead of retrieve-then-mutate, which the SDK silently drops `RequiredLevel` from; the success message (and JSON `requiresPublish`/`publishHint`, MCP `RequiresPublish`/`PublishHint`) now tells the user to run `ppds metadata publish` for the change to take effect (#1009).
- **`ppds custom-apis register` validates nested parameters** — `uniqueName`/`displayName`/`type` on each nested parameter are validated up front with `RequiredField` (naming the offending index), matching `add-parameter`; a null list element is rejected explicitly instead of throwing `NullReferenceException` (#1228, #1230).
- **Text-mode data output is routed to stdout** — ~39 commands across solutions, metadata, plugins, traces, connections, web resources, environments, import jobs, and query history were writing tabular/list data to `stderr`, breaking shell piping and redirection. Data now goes to `stdout`; status, progress, and error messages correctly stay on `stderr` (#1077, #1141).

### Performance
- **`ppds schema compare` parallelizes per-entity metadata fetches** — `EnvironmentSnapshotLoader` replaces its sequential `GetEntityAsync` loop with `Parallel.ForEachAsync` bounded by the pool's recommended parallelism (each task acquires its own pooled client; entity order preserved), cutting snapshot time on large environments (#1139).

## [1.1.0] - 2026-04-26

### Added
- **Web resources** `pull` and `push` commands with content-hash tracking for change detection (#950).
- **`data schema --filter`** option to filter schema generation to specific entities using SQL-like expressions (#502, #948).
- **TUI per-tab profile switching** — individual tabs can target different Dataverse profiles (#927, #935).
- **Diagnostic markers** for SQL and FetchXML syntax errors surfaced via daemon RPC to the extension (#650, #966).
- **Metadata Browser** entity configuration tab and global optionset details in daemon endpoints (#924).
- Unit tests for query models and CLI query commands (#945).
- RPC contract tests for the Extension-CLI daemon protocol (#672, #966).

### Changed
- Per-panel profile and environment scoping — each extension panel tracks its own active context via daemon RPC (#887, #888, #903, #936).

### Fixed
- Solutions panel no longer shows fabricated "All Solutions" entry; component type names auto-synced from metadata (#890, #892, #916).
- Added direct `Microsoft.Data.SqlClient` reference — fixes TDS endpoint assembly-not-found error (#889, #905).
- Sovereign-cloud URL hardcoding in metadata mutation hooks (#870, #947).

## [1.0.0] - 2026-04-18

First stable release. Consolidates features developed across the `1.0.0-beta.1` through `1.0.0-beta.14` series.

### Added

- **Unified CLI (`ppds`)** — Global .NET tool installed via `dotnet tool install -g PPDS.Cli`. Targets `net8.0`, `net9.0`, `net10.0`. Replaces the former `ppds-migrate` tool.
- **Authentication** — `ppds auth create|list|select|delete|update|name|clear|who` with profile storage v2 (name-based active profile, platform-native secure storage). Supports interactive browser, device code, client secret, certificate file/store, managed identity, GitHub/Azure DevOps federated, and username/password. Environment-variable auth via `PPDS_CLIENT_ID` / `PPDS_CLIENT_SECRET` / `PPDS_TENANT_ID` / `PPDS_ENVIRONMENT_URL` for stateless CI/CD ([#706](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/706)). Linux cleartext opt-in via `--accept-cleartext-caching` ([#107](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/107)).
- **Environment management** — `ppds env list|select|who|config` with filtering, URL-based SPN resolution, pre-save WhoAmI validation, configurable label/type/color, and `Open in Maker` / `Open in Dynamics` helpers.
- **Data commands** — `ppds data export|import|copy|analyze|schema|users|load|update|delete|truncate` with bulk APIs, multi-profile pooling, CSV auto-mapping, SQL-like filters, alternate-key upsert, dry-run, and safety guards (`--bypass-plugins`, `--bypass-flows`, `--continue-on-error`, row caps, confirmation prompts). Page-level export parallelism via GUID range partitioning for large entities ([#503](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/503)). Filter feedback surfaced during export ([#501](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/501)).
- **Query commands** — `ppds query fetch|sql|explain` backed by the PPDS.Query engine (ScriptDom parser, FetchXML transpiler, Volcano execution plan). Full T-SQL: SELECT/WHERE/JOIN/GROUP BY/HAVING/ORDER BY, aggregates, subqueries, UNION, CTEs, window functions, variables, IF/ELSE, CASE. DML (INSERT/UPDATE/DELETE) with safety guard and row caps. TDS endpoint routing via `--use-tds`. Cross-environment queries via `[env].[entity]` bracket syntax. Output formats: Text, JSON, CSV ([#52](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/52)).
- **Metadata commands** — `ppds metadata entities|entity|attributes|relationships|keys|optionsets|optionset` for read-only schema browsing, plus `ppds metadata table|column|relationship|choice|key` (with `create|update|delete`) for schema authoring with validation, dry-run, and solution awareness ([#51](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/51), [#764](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/764), [#766](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/766)). `--include-internal` for junction tables; `--include-intersect` for solutions.
- **Plugins commands** — `ppds plugins extract|deploy|diff|list|clean|register|unregister|update|download|get` plus step-level enable/disable and impersonation. `MetadataLoadContext`-based reflection (no assembly load), attribute-driven registration (`PluginStepAttribute`, `PluginImageAttribute`, `CustomApiAttribute`), NuGet plugin-package support, `--secure-config`, `--event-handler-type` for service-endpoint targeting, solution component addition on both create and update ([#464](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/464), [#468](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/468)–[#471](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/471), [#489](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/489), [#490](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/490), [#699](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/699)).
- **Custom APIs commands** — `ppds custom-apis list|get|register|update|unregister|add-parameter|update-parameter|remove-parameter|set-plugin` ([#699](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/699)).
- **Data Providers commands** — `ppds data-providers list|get|register|update|unregister` for virtual-entity providers ([#657](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/657)).
- **Solutions commands** — `ppds solutions list|get|export|import|components|publish|url` ([#137](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/137)).
- **Import Jobs commands** — `ppds importjobs list|get|data|wait|url` ([#138](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/138)).
- **Environment Variables commands** — `ppds environmentvariables list|get|set|export|url` ([#139](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/139)).
- **Users and Roles commands** — `ppds users list|show|roles` and `ppds roles list|show|assign|remove` ([#119](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/119)).
- **Flows, Connections, Connection References** — `ppds flows list|get|url`, `ppds connections list|get`, `ppds connectionreferences list|get|flows|connections|analyze` with orphan detection ([#142](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/142)–[#144](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/144)).
- **Deployment Settings commands** — `ppds deployment-settings generate|sync|validate` (PAC-compatible) ([#145](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/145)).
- **Web Resources commands** — `ppds webresources list|get|url`, top-level `ppds publish --all|web-resources|<name>...`, and entity-type publishing via `ppds publish entity` ([#655](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/655), [#766](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/766)).
- **TUI (`ppds interactive` / `ppds -i`)** — Full Terminal.Gui experience: profile/environment selector with color theming, SQL query editor with syntax highlighting, autocomplete, inline validation, resizable split, multi-tab architecture, query history, cross-environment banner, elapsed-time spinner, `Escape`-cancels. Screens: Solutions, Metadata Browser, Plugin Traces, Web Resources, Connection References, Environment Variables, Migration, Import Jobs, Environment Selector ([#192](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/192), [#434](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/434), [#597](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/597), [#615](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/615)–[#618](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/618), [#708](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/708)). Session persistence for filters per environment per screen ([#288](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/288), [#656](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/656)).
- **Daemon (`ppds serve`)** — JSON-RPC daemon that powers the VS Code extension. Supports all RPC methods for profile, environment, query, metadata, plugin, and panel flows; Information-level log output for troubleshooting.
- **Migration CMT parity** — State transitions (`SetStateRequest` plus Win/Close/Fulfill SDK messages for seven entity types), owner impersonation via `--impersonate-owners`, cascading external lookup resolution, per-entity import-mode overrides, date shifting (4 modes), and file column chunked transfer (4 MB blocks) ([#708](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/708)).
- **Update check system** — `ppds version --check` queries NuGet; background cached check (24 h) with startup notification; `ppds update` for self-update; suppressed via `--quiet` ([#566](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/566)).
- **Global options** — `--quiet`/`-q`, `--verbose`/`-v`, `--debug`, `--correlation-id`, `--output-format`/`-f` (`Text`/`Json`), `--environment` override ([#76](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/76)).
- **Structured error handling** — Hierarchical error codes (`Auth.ProfileNotFound`, `Connection.Failed`, etc.), expanded exit codes (4 `ConnectionError`, 5 `AuthError`, 6 `NotFoundError`), JSON output schema versioning ([#77](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/77)).
- **`ppds docs`** — Opens CLI documentation in browser; documentation URL also surfaced in `--help` ([#165](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/165)).
- **Version header at startup** — CLI, SDK, .NET runtime and platform emitted to stderr for build correlation (skipped for `--help`, `--version`, no args).
- **`ppds logs` command group** — Local-only observability for post-launch triage. `ppds logs tail` shows recent lines with `--lines` and `--level` filtering; `ppds logs dump` bundles logs, `dotnet --info`, and redacted environment variables into a timestamped zip for bug reports ([#862](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/862)).
- **`ppds env config` no-arg mode** — Running `ppds env config` without a URL now shows the active profile's current environment configuration ([#868](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/868)).
- **Progress and status messages route to stderr** — Enables clean `ppds data export -f json | jq` piping without noise in the data stream ([#76](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/76)).
- **`ListResult<T>` transparency** — All list responses expose `Items`, `TotalCount`, `WasTruncated`, and `FiltersApplied`; silent `$top` defaults removed; progressive paging cookie support throughout ([#651](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/651)).
- **Filter UX for metadata commands** — Filters without wildcards perform a contains search (e.g. `--filter zipcode` matches `ppds_zipcode`); `foo*` / `*foo` for explicit prefix/suffix patterns ([#167](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/167)).
- **`ppds flows get|url` accept GUID or unique name** — The `name` argument resolves by workflow ID (GUID) when provided, falling back to unique-name lookup ([#868](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/868)).
- **Extension panel UX** — Unified panel navigation and filtering, 300 ms filter debouncing, and race-condition prevention across 8 VS Code panels ([#633](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/633)).
