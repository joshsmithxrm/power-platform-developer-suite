## [Unreleased]

### Added
- **Canonical metadata nouns** `entity`, `attribute`, `optionset` replace `table`, `column`, `choice` as the primary command surface. Old nouns remain as deprecation shims that delegate to the same execute path and print a migration hint to stderr (#1159).
- **Status reason management** â€” `ppds metadata entity add-statusreason|list-statusreasons|update-statusreason|remove-statusreason` for first-class statuscode option-value management (#1160).
- **`OptionValueDeriver`** shared pure helper for publisher-prefix-based option value derivation â€” explicit-value wins, prefixĂ—10000 base, gap-fill over existing values (#1160).
- **`ppds forms`** â€” form (systemform) management: list/get, set-xml with schema validation, and structured tab/section/field/sub-grid editing on Main forms, with --publish/--solution side effects (#1163).
- **`ppds views`** â€” savedquery (view) management: list/get, add/remove/update/reorder columns, set/clear sort, set/clear filter, and set-fetchxml (#1162).
- **`ppds model-driven-app`** â€” app navigation management: list, get, sitemap, set-sitemap-xml, add-table/remove-table, and set-forms/views/charts, with bundled sitemap XSD validation (#1165).
- **`ppds model-driven-app add-copilot|remove-copilot|list-copilots`** â€” wire a Copilot Studio agent (bot) into an app. The binding is an `appelement` whose polymorphic `objectid` targets the `bot` table, set via the SDK `EntityReference("bot", id)`. The Dataverse Web API cannot create this binding: `appelement.objectid` has three targets (`bot`, `aiskillconfig`, `mcpserver`) that all share the `objectid` navigation property, so a bare `@odata.bind` mis-resolves to `aiskillconfig` and `objectid_bot@odata.bind` is undeclared â€” the SDK carries the explicit target type and has no such ambiguity. `objectid` is create-only (the platform ignores updates to it), so `add-copilot` always creates the binding and falls back to a unique-suffixed `uniquename` when the maker-convention name is already taken by a stale row. Supports `--dry-run` and `--publish`. Note: for the model-driven app designer to surface the agent in-app, the target bot must be a valid app-assistant (`isLightweightBot`); a bot created standalone in Copilot Studio binds correctly at the `appelement` level but won't render until configured as an app assistant (#1185).
- **`ppds model-driven-app inspect-app-assistant`** â€” read-only "copilot doctor" that inspects an app's Copilot (bot) `appelement` bindings and reports problems without writing anything: bots bound but not configured as app assistants (`isLightweightBot=false`, so they never render in-app), duplicate `objectid â†’ bot` bindings for the same app, and orphan copilot-shaped `appelement` rows with a null `objectid`. Each finding reports the bot display name, bot id, `isLightweightBot`, the offending appelement id(s), and a one-line remediation hint. Emits data to stdout (table or `--json`) and status to stderr, and exits non-zero when any problem is found so it is scriptable in CI / pre-publish checks (#1193).
- **`ppds api request`** â€” authenticated raw Web API calls to Dataverse (GET/POST/PATCH/DELETE) with default OData headers and production write protection (#1164).
- **`ppds api request --profile|-p`** â€” target a specific authentication profile, consistent with the other commands (previously `api request` could only switch the environment, not the profile).
- **Local Choice columns** â€” `ppds metadata attribute create --type Choice` with inline --option/--options-file, plus `ppds metadata attribute add-option|update-option|remove-option` for column-scoped option sets (#1161).

### Changed
- `ppds metadata table|column|choice` are now deprecated shims; use `entity|attribute|optionset` instead.

### Fixed
- **Reliable view writes** â€” `ppds views set-fetchxml|set-filter|clear-filter|set-sort|clear-sort|reorder-columns|update-column|remove-column` no longer fail opaquely or report a false success, and now succeed on managed views. Every fetchxml- and layoutxml-mutating command routes through one shared verified write path that (a) surfaces the underlying Dataverse fault (e.g. `0x80040216`) instead of the bare `Details: setting fetchxml`; (b) echoes `returnedtypecode` into the savedquery write so the platform applies fetchxml changes â€” the missing entity context was the actual cause of `0x80040216`; (c) verifies the change persisted via a lag-tolerant read-back against the *unpublished* record (`RetrieveUnpublishedMultiple`), so a draft write is no longer falsely reported as `View.UpdateNotPersisted`; and (d) reserves the `View.ManagedComponentNotEditable` error for genuinely locked components (`iscustomizable = false`) instead of refusing all managed views. Live-verified against a managed Quick Find view (#1190, #1194, #1200, #1201).
- `ppds model-driven-app set-views|set-forms|set-charts` with explicit named components no longer fail with `Input field type 'EntityReference' does not match expected type 'Guid' for field 'AppId'`. The shared `AddAppComponents` helper now passes the appmodule id as a `Guid` (matching the working `RemoveAppComponents` sibling); a regression test asserts the request parameter type (#1183).

## [1.1.0] - 2026-04-26

- **Metadata Browser** entity configuration tab and global optionset details in daemon endpoints (#924).
- Unit tests for query models and CLI query commands (#945).
- RPC contract tests for the Extension-CLI daemon protocol (#672, #966).

### Changed
- Per-panel profile and environment scoping â€” each extension panel tracks its own active context via daemon RPC (#887, #888, #903, #936).

### Fixed
- Solutions panel no longer shows fabricated "All Solutions" entry; component type names auto-synced from metadata (#890, #892, #916).
- Added direct `Microsoft.Data.SqlClient` reference â€” fixes TDS endpoint assembly-not-found error (#889, #905).
- Sovereign-cloud URL hardcoding in metadata mutation hooks (#870, #947).

## [1.0.0] - 2026-04-18

First stable release. Consolidates features developed across the `1.0.0-beta.1` through `1.0.0-beta.14` series.

### Added

- **Unified CLI (`ppds`)** â€” Global .NET tool installed via `dotnet tool install -g PPDS.Cli`. Targets `net8.0`, `net9.0`, `net10.0`. Replaces the former `ppds-migrate` tool.
- **Authentication** â€” `ppds auth create|list|select|delete|update|name|clear|who` with profile storage v2 (name-based active profile, platform-native secure storage). Supports interactive browser, device code, client secret, certificate file/store, managed identity, GitHub/Azure DevOps federated, and username/password. Environment-variable auth via `PPDS_CLIENT_ID` / `PPDS_CLIENT_SECRET` / `PPDS_TENANT_ID` / `PPDS_ENVIRONMENT_URL` for stateless CI/CD ([#706](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/706)). Linux cleartext opt-in via `--accept-cleartext-caching` ([#107](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/107)).
- **Environment management** â€” `ppds env list|select|who|config` with filtering, URL-based SPN resolution, pre-save WhoAmI validation, configurable label/type/color, and `Open in Maker` / `Open in Dynamics` helpers.
- **Data commands** â€” `ppds data export|import|copy|analyze|schema|users|load|update|delete|truncate` with bulk APIs, multi-profile pooling, CSV auto-mapping, SQL-like filters, alternate-key upsert, dry-run, and safety guards (`--bypass-plugins`, `--bypass-flows`, `--continue-on-error`, row caps, confirmation prompts). Page-level export parallelism via GUID range partitioning for large entities ([#503](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/503)). Filter feedback surfaced during export ([#501](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/501)).
- **Query commands** â€” `ppds query fetch|sql|explain` backed by the PPDS.Query engine (ScriptDom parser, FetchXML transpiler, Volcano execution plan). Full T-SQL: SELECT/WHERE/JOIN/GROUP BY/HAVING/ORDER BY, aggregates, subqueries, UNION, CTEs, window functions, variables, IF/ELSE, CASE. DML (INSERT/UPDATE/DELETE) with safety guard and row caps. TDS endpoint routing via `--use-tds`. Cross-environment queries via `[env].[entity]` bracket syntax. Output formats: Text, JSON, CSV ([#52](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/52)).
- **Metadata commands** â€” `ppds metadata entities|entity|attributes|relationships|keys|optionsets|optionset` for read-only schema browsing, plus `ppds metadata table|column|relationship|choice|key` (with `create|update|delete`) for schema authoring with validation, dry-run, and solution awareness ([#51](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/51), [#764](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/764), [#766](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/766)). `--include-internal` for junction tables; `--include-intersect` for solutions.
- **Plugins commands** â€” `ppds plugins extract|deploy|diff|list|clean|register|unregister|update|download|get` plus step-level enable/disable and impersonation. `MetadataLoadContext`-based reflection (no assembly load), attribute-driven registration (`PluginStepAttribute`, `PluginImageAttribute`, `CustomApiAttribute`), NuGet plugin-package support, `--secure-config`, `--event-handler-type` for service-endpoint targeting, solution component addition on both create and update ([#464](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/464), [#468](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/468)â€“[#471](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/471), [#489](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/489), [#490](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/490), [#699](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/699)).
- **Custom APIs commands** â€” `ppds custom-apis list|get|register|update|unregister|add-parameter|update-parameter|remove-parameter|set-plugin` ([#699](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/699)).
- **Data Providers commands** â€” `ppds data-providers list|get|register|update|unregister` for virtual-entity providers ([#657](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/657)).
- **Solutions commands** â€” `ppds solutions list|get|export|import|components|publish|url` ([#137](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/137)).
- **Import Jobs commands** â€” `ppds importjobs list|get|data|wait|url` ([#138](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/138)).
- **Environment Variables commands** â€” `ppds environmentvariables list|get|set|export|url` ([#139](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/139)).
- **Users and Roles commands** â€” `ppds users list|show|roles` and `ppds roles list|show|assign|remove` ([#119](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/119)).
- **Flows, Connections, Connection References** â€” `ppds flows list|get|url`, `ppds connections list|get`, `ppds connectionreferences list|get|flows|connections|analyze` with orphan detection ([#142](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/142)â€“[#144](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/144)).
- **Deployment Settings commands** â€” `ppds deployment-settings generate|sync|validate` (PAC-compatible) ([#145](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/145)).
- **Web Resources commands** â€” `ppds webresources list|get|url`, top-level `ppds publish --all|web-resources|<name>...`, and entity-type publishing via `ppds publish entity` ([#655](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/655), [#766](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/766)).
- **TUI (`ppds interactive` / `ppds -i`)** â€” Full Terminal.Gui experience: profile/environment selector with color theming, SQL query editor with syntax highlighting, autocomplete, inline validation, resizable split, multi-tab architecture, query history, cross-environment banner, elapsed-time spinner, `Escape`-cancels. Screens: Solutions, Metadata Browser, Plugin Traces, Web Resources, Connection References, Environment Variables, Migration, Import Jobs, Environment Selector ([#192](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/192), [#434](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/434), [#597](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/597), [#615](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/615)â€“[#618](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/618), [#708](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/708)). Session persistence for filters per environment per screen ([#288](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/288), [#656](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/656)).
- **Daemon (`ppds serve`)** â€” JSON-RPC daemon that powers the VS Code extension. Supports all RPC methods for profile, environment, query, metadata, plugin, and panel flows; Information-level log output for troubleshooting.
- **Migration CMT parity** â€” State transitions (`SetStateRequest` plus Win/Close/Fulfill SDK messages for seven entity types), owner impersonation via `--impersonate-owners`, cascading external lookup resolution, per-entity import-mode overrides, date shifting (4 modes), and file column chunked transfer (4 MB blocks) ([#708](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/708)).
- **Update check system** â€” `ppds version --check` queries NuGet; background cached check (24 h) with startup notification; `ppds update` for self-update; suppressed via `--quiet` ([#566](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/566)).
- **Global options** â€” `--quiet`/`-q`, `--verbose`/`-v`, `--debug`, `--correlation-id`, `--output-format`/`-f` (`Text`/`Json`), `--environment` override ([#76](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/76)).
- **Structured error handling** â€” Hierarchical error codes (`Auth.ProfileNotFound`, `Connection.Failed`, etc.), expanded exit codes (4 `ConnectionError`, 5 `AuthError`, 6 `NotFoundError`), JSON output schema versioning ([#77](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/77)).
- **`ppds docs`** â€” Opens CLI documentation in browser; documentation URL also surfaced in `--help` ([#165](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/165)).
- **Version header at startup** â€” CLI, SDK, .NET runtime and platform emitted to stderr for build correlation (skipped for `--help`, `--version`, no args).
- **`ppds logs` command group** â€” Local-only observability for post-launch triage. `ppds logs tail` shows recent lines with `--lines` and `--level` filtering; `ppds logs dump` bundles logs, `dotnet --info`, and redacted environment variables into a timestamped zip for bug reports ([#862](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/862)).
- **`ppds env config` no-arg mode** â€” Running `ppds env config` without a URL now shows the active profile's current environment configuration ([#868](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/868)).
- **Progress and status messages route to stderr** â€” Enables clean `ppds data export -f json | jq` piping without noise in the data stream ([#76](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/76)).
- **`ListResult<T>` transparency** â€” All list responses expose `Items`, `TotalCount`, `WasTruncated`, and `FiltersApplied`; silent `$top` defaults removed; progressive paging cookie support throughout ([#651](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/651)).
- **Filter UX for metadata commands** â€” Filters without wildcards perform a contains search (e.g. `--filter zipcode` matches `ppds_zipcode`); `foo*` / `*foo` for explicit prefix/suffix patterns ([#167](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/167)).
- **`ppds flows get|url` accept GUID or unique name** â€” The `name` argument resolves by workflow ID (GUID) when provided, falling back to unique-name lookup ([#868](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/868)).
- **Extension panel UX** â€” Unified panel navigation and filtering, 300 ms filter debouncing, and race-condition prevention across 8 VS Code panels ([#633](https://github.com/joshsmithxrm/power-platform-developer-suite/issues/633)).
